GETTING STARTED

Thiết lập môi trường phát triển

Để biên dịch và chạy thử nghiệm WinNest locally, bạn cần cài đặt **Node.js LTS (>=v20)** và hệ thống chạy **Wine** (bao gồm `wine` và `wineserver`).

Khuyến nghị hệ thống

Nên sử dụng Wine phiên bản 8.0 trở lên để có khả năng tương thích API tốt nhất với các phần mềm Windows hiện đại.

Sử dụng các câu lệnh sau để cài đặt dependencies, kiểm tra kiểu tĩnh và build mã nguồn:

Chức năng Câu lệnh CLI
Cài đặt node modules npm install
Kiểm tra kiểu tĩnh (TypeScript) npm run check
Biên dịch sang JavaScript npm run build
Kiểm tra lỗi đường dẫn phức tạp npm run test:path-torture
CLI REFERENCE

Tham chiếu câu lệnh dòng lệnh

WinNest cung cấp các câu lệnh quản lý toàn bộ vòng đời của ứng dụng:

winnest doctor

Kiểm tra trạng thái hệ thống: Xác minh sự hiện diện của trình chạy Wine, thư mục shortcut desktop, và kiểm tra tính hợp lệ của phiên bản Node.js hiện hành.

winnest install <installer-path>

Tiến hành khởi tạo vùng chứa (sandbox) mới cho app. Trình cài đặt sẽ được sao chép vào bộ nhớ đệm, sau đó chạy Wine để thực hiện cài đặt. Sau khi cài đặt hoàn tất, WinNest phân tích các phím tắt được ứng dụng tạo ra để ánh xạ sang desktop launcher.

winnest run <app-id>

Vận hành ứng dụng đã cài đặt. Lệnh này sẽ tải đúng tệp cấu hình, thiết lập môi trường prefix tương ứng, chuyển tiếp luồng log và gọi tiến trình ứng dụng.

winnest open <file-path>

Mở một tệp tin thông qua trình chạy WinNest tích hợp. Đây là lối vào để liên kết định dạng tệp tin hệ thống (MIME type) với các ứng dụng chạy qua Wine.

winnest reset <app-id>

Xóa bỏ toàn bộ dữ liệu người dùng của ứng dụng Windows, đặt lại registry và cấu hình Wine prefix về trạng thái ban đầu của bộ cài đặt sạch.

winnest uninstall <app-id>

Dỡ bỏ hoàn toàn ứng dụng khỏi WinNest. Xóa thư mục chứa Wine prefix, tệp cấu hình launcher, các file log, và các shortcut `.desktop` liên kết.

LAYOUT SPECIFICATION

Cấu trúc thư mục dữ liệu

Tất cả các dữ liệu hoạt động của WinNest được lưu trữ tập trung tại thư mục người dùng Linux:

~/.local/share/winnest/apps/<app-id>/
├── app.json       # Metadata (Tên ứng dụng, phiên bản, ID, đường dẫn exe gốc)
├── prefix/         # Thư mục Wine Prefix cô lập hoàn toàn
│   └── drive_c/     # Ổ đĩa ảo C: của Windows ảo
├── installer/      # File cài đặt gốc dạng sao lưu để tái thiết lập
└── logs/           # Các file ghi nhật ký hoạt động có cấu trúc dạng JSON-lines

Định dạng file app.json

Mỗi ứng dụng lưu giữ cấu hình cấu trúc JSON chuẩn hóa dưới đây để xác định hành vi chạy:

app.json 
{
  "id": "steam",
  "name": "Steam Client",
  "version": "1.0.0",
  "installedAt": "2026-07-08T00:00:00.000Z",
  "executables": [
    {
      "name": "steam.exe",
      "path": "Program Files (x86)/Steam/steam.exe",
      "args": ["-silent"]
    }
  ]
}
SECURITY & ARCHITECTURE

Mô hình an toàn bảo mật

WinNest được thiết kế với nguyên lý "Không tin tưởng mã cài đặt Windows". Các tệp tin cài đặt exe hoặc msi có thể chứa các đoạn mã độc hại hoặc hành vi không kiểm soát. Do đó, chúng tôi áp dụng hai rào cản bảo vệ:

Vô hiệu hóa thực thi Shell thô

Hệ thống Core sử dụng phương thức khởi chạy an toàn `spawn(command, args)` thay vì chạy qua `exec` của Shell Linux. Điều này ngăn chặn triệt để tấn công chèn lệnh (Command Injection) từ các đường dẫn tệp tin chứa ký tự đặc biệt.

Giới hạn truy cập Prefix

Mặc dù ứng dụng Windows trong Wine vẫn có quyền đọc file của user nếu Wine map thư mục home, việc cô lập phân vùng registry và ổ đĩa C: giúp giảm thiểu tối đa nguy cơ lây nhiễm chéo hoặc can thiệp sâu vào nhân Linux.
